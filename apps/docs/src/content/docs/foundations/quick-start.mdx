The quickest way to see Chamfer UI in action is to install the core CSS bundle, spin up the HTML playground, and render the button component. The steps below assume you’re using pnpm, but the commands translate to npm/yarn.

## 1. Install packages

```
# pnpm
pnpm add @chamfer/core @chamfer/behavior

# npm
npm install @chamfer/core @chamfer/behavior

# yarn
yarn add @chamfer/core @chamfer/behavior
```

Need Tailwind utilities? Install `@chamfer/tailwind` alongside the core packages.

## 2. Import baseline CSS

```html
<!-- app.html -->
<link rel="stylesheet" href="/node_modules/@chamfer/core/dist/css/index.css" />
```

When bundling with Vite/Next/etc. you can import the module path directly:

```ts
// main.ts
import "@chamfer/core/css";
```

_Node 18.18+ or 20.9+ is recommended. Vite/Next/Remix all handle the CSS import without extra configuration._

## 3. Render a button

```html
<button class="ch-button ch-button--md ch-button--solid ch-button--primary" data-ch-component="button">
  <span class="ch-button__label">Save changes</span>
</button>
```

To enable ripple behavior, call the helper from `@chamfer/behavior`:

```ts
import { enhanceButton } from "@chamfer/behavior";

document.querySelectorAll('[data-ch-component="button"]').forEach(element => {
  enhanceButton(element as HTMLButtonElement);
});
```

**React example**

```tsx
import "@chamfer/core/css";
import { useEffect, useRef } from "react";
import { enhanceButton } from "@chamfer/behavior";

export function SaveButton() {
  const ref = useRef<HTMLButtonElement>(null);

  useEffect(() => {
    if (!ref.current) return;
    const enhancement = enhanceButton(ref.current);
    return enhancement.destroy;
  }, []);

  return (
    <button ref={ref} className="ch-button ch-button--md ch-button--solid ch-button--primary" data-ch-component="button">
      <span className="ch-button__label">Save changes</span>
    </button>
  );
}
```

## 4. Tailwind integration (optional)

```ts
// tailwind.config.ts
import { chamferTailwindPreset } from "@chamfer/tailwind";

export default {
  presets: [chamferTailwindPreset]
};
```

Now you can use utilities such as `bg-ch-action-primary`, `px-ch-button-inline-md`, and `focus-ring-ch` that stay in sync with the underlying tokens.

## 5. Explore the playground

The repo ships an HTML playground showcasing tokens and components. From the root of the monorepo:

```bash
pnpm --filter @chamfer/playground-html dev
```

This launches a Vite server with live reload. Rebuild tokens/core (`pnpm --filter @chamfer/tokens build` then `pnpm --filter @chamfer/core build`) before starting if you’ve edited token sources.
