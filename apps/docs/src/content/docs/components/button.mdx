## Overview

Buttons trigger immediate actions in the current view. Chamfer exposes semantic tone tokens and a density scale so the same CSS covers interactive elements across surfaces.

**Do**
- Drive a primary outcome such as Save, Publish, or Confirm with the solid primary tone.
- Pair a leading or trailing icon to clarify meaning while keeping a visible text label.
- Use the `flat` emphasis on tinted or elevated surfaces where `soft` feels too heavy.
- Disable or show a loading state while asynchronous work is pending.

**Do not**
- Use buttons for navigation that changes location; prefer an anchor styled as a button.
- Invent custom colors that bypass semantic tokens; map to an existing tone instead.
- Hide focus indicators or rely on color alone to indicate destructive actions.

## Anatomy

```html
<button class="ch-button ch-button--md ch-button--solid ch-button--primary" data-ch-component="button" type="button">
  <span class="ch-button__leading" aria-hidden="true"></span>
  <span class="ch-button__label">Label</span>
  <span class="ch-button__trailing" aria-hidden="true"></span>
</button>
```

| Element or attribute | Purpose |
| --- | --- |
| `<button>` | Preferred host element; use `<a role="button">` when navigation is required. |
| `data-ch-component="button"` | Required for styling hooks and automated tests. |
| `.ch-button__leading` | Optional icon or badge slot; hide with `hidden` when unused. |
| `.ch-button__label` | Visible text that forms the accessible name. |
| `.ch-button__trailing` | Optional icon or status indicator. |

## States

- Hover and active states derive from emphasis tokens; the preview below uses `data-demo-state` helpers.
- Focus visible uses Chamfer focus ring tokens; never remove focus outlines.
- Disabled state relies on the native `disabled` attribute or `aria-disabled="true"` for anchors.
- Loading uses `data-ch-loading="true"`, `aria-busy="true"`, and blocks double submits. A spinner will ship as a separate component.
- High contrast (forced colors) uses `--ch-button-hcm-bg` and `--ch-button-hcm-fg`.

## Behavior

### Using `enhanceButton`

Chamfer buttons work with pure HTML and CSS. When you need animated feedback or keyboard polish, import `enhanceButton` and apply it to the elements you render. The helper stamps required data attributes, ensures a `type="button"` default on native buttons, and wires up the ripple effect when enabled. The demo below includes both the HTML markup and the JavaScript helper sources.

<DemoBlock demo={rippleDemo} component={RippleDemo} />

### Options

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `ripple` | `boolean` | `true` (unless `data-ch-ripple="false"` is present) | Toggles the ripple wave. Set to `false` when motion should be suppressed or the surface already provides feedback. |

### Guidelines

- Disable ripple with `data-ch-ripple="false"` when visual feedback is undesirable or when rendering inside motion-reduced contexts.
- Always keep native focus outlines; enhanceButton augments keyboard handling but does not remove focus rings.
- For anchors, keep `role="button"` and `tabindex=0` so the helper can apply the same semantics.
- The helper is progressive enhancement; if JavaScript is unavailable the button remains fully usable.

## Accessibility

| Interaction | Expectation |
| --- | --- |
| Tab or Shift+Tab | Moves focus in and out of buttons following DOM order. |
| Enter or Space | Activates the button; Space triggers press sequence on keyup. |
| Escape | Dismisses enclosing dialogs when the button closes them. |

Best practices:

- Icon only buttons must set `aria-label` or `aria-labelledby`.
- Use `aria-live` or toast patterns to announce results of asynchronous actions.
- Ensure disabled and loading states are communicated with `disabled`, `aria-disabled`, and `aria-busy`.
- Test with screen readers (VoiceOver, NVDA) and automated tools (axe, Pa11y).

## Reference

### Classes

| Class | Purpose |
| --- | --- |
| `ch-button` | Base styling and layout. |
| `ch-button--{sm\|md\|lg}` | Density and typography scale. |
| `ch-button--{solid\|soft\|flat\|outline\|ghost}` | Emphasis variants. |
| `ch-button--{primary\|secondary\|success\|warning\|danger}` | Semantic tone. |
| `ch-button--icon-only` | Icon only layout with hidden label. |

### Data attributes

| Attribute | Values | Effect |
| --- | --- | --- |
| `data-ch-component` | `"button"` | Required identifier for styling and testing hooks. |
| `data-ch-loading` | `"true"` | Applies loading visuals and pointer suppression. |
| `data-ch-ripple` | `"false"` | Disables ripple for specific buttons. |
| `data-demo-state` | `hover`, `active`, `focus` | Documentation helper only. |

### Tokens

| Token | Description |
| --- | --- |
| `--ch-action-<tone>-bg` | Solid background for each semantic tone. |
| `--ch-action-<tone>-foreground` | Text color that satisfies contrast targets. |
| `--ch-button-padding-inline-[sm\|md\|lg]` | Density based horizontal padding. |
| `--ch-pressable-height-[sm\|md\|lg]` | Control height per size tier. |
| `--ch-button-radius` | Corner radius shared across pressables. |
| `--ch-ghost-hover-base` and `--ch-ghost-active-base` | Overlay strengths used by flat and ghost. |
| `--ch-focus-ring-inner` and `--ch-focus-ring-outer` | Focus halo colors. |

## Content guidelines

- Use concise imperative verbs (Save, Publish, Delete).
- Keep labels under 24 characters; allow wrapping rather than truncation.
- Pair icons with text except when space constrained; icon only buttons must include `aria-label`.
- For destructive actions, use danger tone plus explicit copy (Delete project).

## Testing

Selectors:

- Root: `[data-ch-component="button"]`
- Loading: `[data-ch-component="button"][data-ch-loading="true"]`

Playwright:

```ts
await page.getByRole("button", { name: "Save changes" }).click();
await expect(page.getByRole("button", { name: "Save changes" })).toHaveAttribute("aria-busy", "true");
```

Axe smoke test:

```ts
const results = await axe(document.body);
expect(results.violations).toHaveLength(0);
```

## Performance

- Base button is zero JavaScript; core CSS covers all states.
- Avoid excessive drop shadows or blur filters on mobile to keep GPU compositing light.

## Internationalization and RTL

- Labels wrap by default; confirm long translations (for example German) do not overflow containers.
- Icon slots automatically mirror in RTL because flexbox reverses direction; verify custom trailing content.
- Use sentence case and avoid all caps to keep readability in languages without casing.
