Chamfer buttons are the canonical call-to-action component. They lean entirely on semantic tokens (tone, emphasis, density) so brands can theme the system without rewriting CSS. Use them for form submissions, navigation actions, or contextual operations that need a visible affordance.

## When to use

- Promote high-impact actions such as “Save”, “Publish”, or “Delete”.
- Pair label text with optional leading/trailing icons while keeping an accessible name.
- Reflect semantic meaning through the provided tones instead of inventing ad hoc utilities.

## Install & import

```
# pnpm
pnpm add @chamfer/core @chamfer/behavior

# npm
npm install @chamfer/core @chamfer/behavior

# yarn
yarn add @chamfer/core @chamfer/behavior
```

```ts
// main.ts
import "@chamfer/core/css";
```

## Anatomy & tokens

- Root: `<button>` (or `<a role="button">`) with `data-ch-component="button"`.
- Slots: `.ch-button__leading`, `.ch-button__label`, `.ch-button__trailing` align icon and text.
- Loading state: set `data-ch-loading="true"` + `disabled` + `aria-busy="true"` to suspend interaction.

| Token | Description |
| --- | --- |
| `--ch-button-padding-inline-[sm\|md\|lg]` | Density-aware inline padding |
| `--ch-button-padding-block` | Baseline-aligned vertical padding |
| `--ch-button-gap-inline` | Space between leading, label, and trailing slots |
| `--ch-button-radius`, `--ch-button-radius-lg` | Radius scale per size tier |
| `--ch-button-icon-size-[sm\|md\|lg]` | Icon dimensions mapped to semantic icon tokens |
| `--ch-button-font-weight` | Label weight (defaults to `--ch-weight-medium`) |
| `--ch-button-hcm-*` | Forced-color palette for Windows High Contrast |

Emphasis colours come from `--ch-action-<tone>-*` tokens; ghost emphasis uses `--ch-ghost-hover` / `--ch-ghost-active` overlays.

## Variants & sizes

| Emphasis | Semantic tokens | Notes |
| --- | --- | --- |
| Solid | `--ch-action-<tone>-bg/fg/hover/active` | High-contrast primary CTA |
| Soft | `--ch-action-<tone>-soft-*` | Subtle tone for secondary actions |
| Outline | `--ch-action-outline-<tone>-*` | Transparent fill with tokenised borders |
| Ghost | `--ch-ghost-hover(-base|raised)` | Overlay approach for neutral surfaces |

| Size modifier | Height token | Inline padding | Font size |
| --- | --- | --- | --- |
| `ch-button--sm` | `--ch-pressable-height-sm` | `--ch-button-padding-inline-sm` | `--ch-text-sm` |
| `ch-button--md` | `--ch-pressable-height-md` | `--ch-button-padding-inline-md` | `--ch-text-md` |
| `ch-button--lg` | `--ch-pressable-height-lg` | `--ch-button-padding-inline-lg` | `--ch-text-lg` |

Add `ch-button--icon-only` for square icon buttons and supply an `aria-label`.

## API reference

| Modifier | Purpose |
| --- | --- |
| `ch-button--{sm\|md\|lg}` | Adjusts density and typography |
| `ch-button--{solid\|soft\|outline\|ghost}` | Sets emphasis style |
| `ch-button--{primary\|secondary\|success\|warning\|danger}` | Applies semantic tone |
| `ch-button--icon-only` | Hides visual label; supply `aria-label` |
| `data-ch-ripple="false"` | Disables ripple enhancement |
| `data-ch-ripple="raised"` | Uses raised-surface ripple overlay |
| `data-ch-loading="true"` | Indicates loading/busy state |

## Accessibility

- Provide accessible names (`aria-label` for icon-only buttons).
- Respect `disabled` or `aria-disabled="true"` + `tabindex="-1"` semantics.
- Set `aria-busy="true"` when loading and keep the control disabled.
- Focus styles use tokenised outlines ensuring 3:1 contrast; verify in high-contrast mode.
- Ripple enhancement honours `prefers-reduced-motion` automatically.

See the [Accessibility baseline](/foundations/accessibility) for the project-wide checklist.
