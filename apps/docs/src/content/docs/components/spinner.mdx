## Overview

The Spinner is an indeterminate loading indicator that communicates async progress to users. It supports semantic tone variants (primary, secondary, success, warning, danger, info), three sizes (sm, md, lg), and respects `prefers-reduced-motion` for accessibility.

**Do**
- Use spinners for operations with unknown duration (file uploads, API calls).
- Pair with descriptive text or `aria-label` to provide context.
- Combine with `aria-busy="true"` on the container to announce loading state.
- Disable button interactions while async work is pending.

**Do not**
- Use spinners for determinate progress; use a progress bar instead.
- Rely on color alone to indicate status; always pair with text or ARIA labels.
- Ignore `prefers-reduced-motion: reduce`; pause or significantly slow animations for users who prefer no motion.

## Usage

Chamfer ships spinners as plain HTML and CSS. Use the interactive preview to see default styling, then copy the minimal markup to adapt it to your needs.

### Minimal HTML

```html
<div
  class="ch-spinner ch-spinner--md ch-spinner--primary"
  data-ch-component="spinner"
  aria-label="Loading"
  role="status"
>
  <svg viewBox="0 0 24 24" class="ch-spinner__icon">
    <circle cx="12" cy="12" r="10" fill="none" stroke="currentColor" stroke-width="2" stroke-dasharray="62.83 62.83" />
  </svg>
</div>
```

## Anatomy

| Element or attribute | Purpose |
| --- | --- |
| `.ch-spinner` | Root container for the spinner. |
| `data-ch-component="spinner"` | Required for styling hooks and automated tests. |
| `.ch-spinner__icon` | SVG circle that rotates to indicate loading. |
| `role="status"` | Announces the spinner as a live status region. |
| `aria-label` | Provides accessible context when no visible label is present. |

## Variants

### Tone scale

| Tone | Token family | Typical use |
| --- | --- | --- |
| `ch-spinner--primary` | `--ch-tone-primary-fg` | Default loading indicator. |
| `ch-spinner--secondary` | `--ch-tone-secondary-fg` | Secondary context or less urgent loading. |
| `ch-spinner--success` | `--ch-tone-success-fg` | Success or completion feedback. |
| `ch-spinner--warning` | `--ch-tone-warning-fg` | Warning or caution feedback. |
| `ch-spinner--danger` | `--ch-tone-danger-fg` | Danger or error feedback. |
| `ch-spinner--info` | `--ch-tone-info-fg` | Informational or neutral context. |

### Size scale

| Size modifier | Diameter token | Typical use |
| --- | --- | --- |
| `ch-spinner--sm` | `--ch-icon-size-sm` | Compact inline loading (e.g., button loading state). |
| `ch-spinner--md` | `--ch-icon-size-md` | Standard full-page loading overlay. |
| `ch-spinner--lg` | `--ch-icon-size-lg` | Large loading screen or hero section. |

## States

- **Default:** Continuous rotation animating at 1 second per cycle.
- **Paused:** Set `data-ch-paused="true"` to halt animation (useful for testing or explicit UI control).
- **Reduced motion:** Respects `prefers-reduced-motion: reduce` by slowing animation to 2 seconds and using stepped keyframes.
- **High contrast (forced colors):** Uses `CanvasText` for stroke color to ensure visibility in Windows High Contrast modes.

## Behavior

### Animation

The spinner uses a CSS `@keyframes` animation to rotate 360° continuously. Animation timing is:
- **Default:** 1 second, linear easing (no acceleration).
- **Reduced motion:** 2 seconds, stepped easing for smoother transitions with `prefers-reduced-motion: reduce`.

### Accessibility

- Provide `aria-label` or place the spinner within a container with `aria-busy="true"` and surrounding descriptive text.
- Use `role="status"` to announce the spinner as a live region (screen readers will announce changes).
- Respect `prefers-reduced-motion: reduce` automatically; the spinner will slow down or use stepped animation.
- Ensure stroke color meets 3:1 contrast against the adjacent background.

## Theming tokens

| Token | Purpose |
| --- | --- |
| `--ch-tone-<tone>-fg` | Spinner stroke color, inherited via `currentColor`. |
| `--ch-icon-size-sm \| md \| lg` | Spinner diameter for each size variant. |
| `--ch-spinner-animation-duration` | Customizable animation duration (default: 1s). |
| `--ch-spinner-animation-timing` | Customizable easing function (default: linear). |

### Custom color example

```css
/* Override spinner color */
.ch-spinner--primary {
  --ch-spinner-color: #3b82f6;
}

/* Or use inline style */
<div class="ch-spinner ch-spinner--md" style="--ch-spinner-color: #ef4444;">
```

## API contract

### CSS classes

- `.ch-spinner` – Root component class.
- `.ch-spinner--sm` – Small size variant.
- `.ch-spinner--md` – Medium size variant (default).
- `.ch-spinner--lg` – Large size variant.
- `.ch-spinner--primary` – Primary tone variant (default).
- `.ch-spinner--secondary` – Secondary tone variant.
- `.ch-spinner--success` – Success tone variant.
- `.ch-spinner--warning` – Warning tone variant.
- `.ch-spinner--danger` – Danger tone variant.
- `.ch-spinner--info` – Info tone variant.
- `.ch-spinner__icon` – SVG container class.
- `.ch-spinner__label` – Optional label container (hidden by default).

### Data attributes

- `data-ch-component="spinner"` – Required component identifier.
- `data-ch-paused="true"` – Pauses animation (optional).

## Guidelines

### Context and labeling

Always provide accessible context:

```html
<!-- With aria-label -->
<div class="ch-spinner ch-spinner--md ch-spinner--primary" aria-label="Uploading file..." role="status"></div>

<!-- With surrounding text -->
<div aria-busy="true">
  <div class="ch-spinner ch-spinner--md ch-spinner--primary"></div>
  <p>Processing your request...</p>
</div>
```

### Button loading state

Pair with a button to indicate async work:

```html
<button class="ch-button ch-button--md ch-button--primary" aria-busy="true" disabled>
  <span class="ch-spinner ch-spinner--sm ch-spinner--primary" aria-hidden="true"></span>
  <span>Loading...</span>
</button>
```

### Full-page overlay

Use a larger spinner for full-page loading:

```html
<div class="loading-overlay" aria-busy="true">
  <div class="ch-spinner ch-spinner--lg ch-spinner--primary" aria-label="Loading..." role="status"></div>
</div>
```

## Testing

### Selectors

- `[data-ch-component="spinner"]` – Query any spinner.
- `.ch-spinner--sm .ch-spinner--md .ch-spinner--lg` – Query by size.
- `.ch-spinner--primary` – Query by tone.

### Example test

```js
// Check that spinner is paused
const spinner = screen.getByRole("status");
expect(spinner).toHaveAttribute("data-ch-paused", "true");

// Check accessible label
expect(spinner).toHaveAccessibleName("Loading...");
```

## Performance

- CSS animations are GPU-accelerated; no JavaScript required.
- Spinner rotation uses `transform: rotate()` for optimal performance.
- `prefers-reduced-motion: reduce` automatically optimizes animation for users with vestibular disorders.

## Internationalization and RTL

- Spinners rotate identically in LTR and RTL layouts.
- Always pair with translatable text via `aria-label` or surrounding content.
