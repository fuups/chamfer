Overview

Vanilla HTML rendering of the Chamfer button. The root element uses the ch-button class and data-ch-component="button" attribute.

Specification reference: components/button.md

Sizes & slots

States & behaviors

Hover/active/focus states are forced for illustration using data-demo-state; interact with the default buttons to observe real transitions. The loading state disables the ripple and sets aria-busy="true". Use the high contrast toggle in the header to simulate the upcoming .hcm mode.

Semantics

Anchor role=button Disabled link

Non-<button> elements should set an explicit role, keyboard focus, and aria-disabled/tabindex handling. The core styles are applied via class="ch-button" plus data-ch-component="button".

RTL & long labels

Buttons maintain slot ordering and padding in right-to-left layouts. Long labels wrap without collapsing the leading/trailing content.
